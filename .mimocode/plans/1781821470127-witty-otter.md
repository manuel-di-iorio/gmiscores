# Piano: Sistema di Team

## Contesto
La piattaforma GMI Scores è un'app PHP per gestire classifiche di giochi. Attualmente ogni utente gestisce i propri giochi in modo indipendente. Serve introdurre un sistema di team per permettere a più utenti di collaborare sui medesimi giochi.

## Database

### Migration 005: Tabelle team

```sql
-- Tabella teams
CREATE TABLE teams (
  team_id INT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(255) NOT NULL,
  created_by INT NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (created_by) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- Tabella team_members
CREATE TABLE team_members (
  id INT AUTO_INCREMENT PRIMARY KEY,
  team_id INT NOT NULL,
  user_id INT NOT NULL,
  role ENUM('admin','member') NOT NULL DEFAULT 'member',
  added_by INT NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY unique_team_user (team_id, user_id),
  FOREIGN KEY (team_id) REFERENCES teams(team_id) ON DELETE CASCADE,
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
  FOREIGN KEY (added_by) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### Migration 006: Colonna team_id su games

```sql
ALTER TABLE games ADD COLUMN team_id INT DEFAULT NULL AFTER user_id;
ALTER TABLE games ADD FOREIGN KEY (team_id) REFERENCES teams(team_id) ON DELETE SET NULL;
```

## Modelli

### `models/Team.php` (nuovo)
- `create(int $userId, string $name)` - Crea team, aggiunge il creatore come admin
- `listByUser(int $userId)` - Lista team di cui l'utente è membro
- `getById(int $teamId)` - Dettaglio team
- `updateName(int $teamId, int $userId, string $name)` - Rinomina (solo admin)
- `delete(int $teamId, int $userId)` - Elimina team (solo admin + creator)
- `addMember(int $teamId, int $addedByUserId, string $discordUserId, string $role)` - Aggiunge membro tramite Discord ID
- `removeMember(int $teamId, int $removedByUserId, int $targetUserId)` - Rimuove membro (non se stesso)
- `getMembers(int $teamId)` - Lista membri
- `isMember(int $teamId, int $userId)` - Controlla membership
- `isAdmin(int $teamId, int $userId)` - Controlla se admin
- `getByUser(int $userId)` - Tutti i team dell'utente con dettagli
- `countMembers(int $teamId)` - Conta membri

### `models/Game.php` (modifiche)
- `create(int $userId, string $gameName, string $clientSecret, ?int $teamId = NULL)` - Aggiunto parametro team_id
- `listByTeam(int $teamId, ?string $nameFilter = null)` - Lista giochi di un team
- `moveToTeam(int $gameId, int $userId, ?int $targetTeamId)` - Sposta gioco in un altro team
- `getByIdAndTeam(int $gameId, int $userId)` - Accesso al gioco via team membership
- Modificare `getByIdAndUser` per supportare anche accesso via team

## Pagine

### `teams.php` - Lista team
- Mostra team di cui l'utente è membro
- Pulsante "Crea team"
- Per ogni team: nome, ruolo, numero membri, link a dettaglio

### `team.php?id=X` - Dettaglio team
- Tab Config: nome team, modifica nome
- Tab Membri: lista membri con ruolo, opzioni aggiungi/rimuovi
- Tab Giochi: lista giochi del team con opzione sposta
- Pulsante elimina team (solo admin, con conferma)

### `add-team.php` - Crea team
- Form con campo nome
- POST → crea team + aggiunge creatore come admin

### `team-settings.php?id=X` - Modifica team
- POST: rinomina team
- DELETE: elimina team

### `team-members.php?id=X` - Gestione membri
- POST: aggiunge membro (discord_id, ruolo)
- DELETE: rimuove membro

### `team-move-game.php?id=X` - Sposta gioco
- GET: mostra form con select team destination
- POST: sposta gioco
